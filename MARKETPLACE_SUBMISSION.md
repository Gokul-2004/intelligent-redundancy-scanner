# Google Workspace Marketplace Submission Checklist

## Pre-Submission Requirements (100% Approval Strategy)

### 1. Technical Requirements ✅

- [x] Google Workspace Add-on built with Apps Script
- [x] Card Service UI implemented
- [x] OAuth 2.0 properly configured
- [x] Backend API deployed and accessible
- [x] HTTPS/SSL enabled
- [x] Error handling implemented
- [x] Rate limiting configured

### 2. OAuth Verification ✅

- [ ] Privacy Policy created and hosted
- [ ] Terms of Service created and hosted
- [ ] OAuth consent screen configured
- [ ] All required scopes added
- [ ] Test users added
- [ ] OAuth verification video created (3-5 minutes)
- [ ] Video uploaded to YouTube (unlisted)
- [ ] Video URL submitted in OAuth consent screen

### 3. Documentation ✅

- [ ] Privacy Policy (must cover):
  - What data is collected
  - How data is used
  - Data storage (stateless - no storage)
  - Data sharing (none)
  - User rights
  - Contact information
- [ ] Terms of Service (must cover):
  - Service description
  - User responsibilities
  - Limitation of liability
  - Intellectual property
  - Contact information
- [ ] Support documentation:
  - How to use the add-on
  - Troubleshooting guide
  - FAQ

### 4. Store Listing Materials ✅

- [ ] App name: "Intelligent Redundancy Scanner"
- [ ] Short description (80 characters max)
- [ ] Long description (4000 characters max)
- [ ] Category: Productivity / Storage Management
- [ ] App icon (512x512px PNG)
- [ ] Screenshots (at least 5, 1280x800px):
  - Homepage card
  - Folder selection
  - Scanning progress
  - Results display
  - Duplicate groups
- [ ] Promotional video (optional but recommended)
- [ ] Support email
- [ ] Privacy policy URL
- [ ] Terms of service URL

### 5. Security & Compliance ✅

- [ ] Data handling clearly documented
- [ ] No sensitive data stored
- [ ] Secure API communication (HTTPS)
- [ ] OAuth tokens handled securely
- [ ] Error messages don't expose sensitive info
- [ ] Rate limiting to prevent abuse

### 6. Testing ✅

- [ ] Tested in Google Drive
- [ ] Tested folder selection
- [ ] Tested scanning process
- [ ] Tested results display
- [ ] Tested error handling
- [ ] Tested with multiple users
- [ ] Tested with large folders
- [ ] Performance tested

### 7. Marketplace SDK Configuration ✅

- [ ] Google Workspace Marketplace SDK enabled
- [ ] Store listing created
- [ ] All required fields filled
- [ ] Screenshots uploaded
- [ ] Pricing set (Free recommended for initial launch)
- [ ] Support information added

## Submission Process

### Step 1: Complete OAuth Verification (2-4 weeks)

1. Create privacy policy and terms
2. Record OAuth demo video
3. Upload video to YouTube (unlisted)
4. Submit OAuth consent screen for verification
5. Wait for Google review
6. Address any feedback

### Step 2: Prepare Store Listing (1 week)

1. Create all marketing materials
2. Write descriptions
3. Take screenshots
4. Design app icon
5. Prepare promotional video (optional)

### Step 3: Submit to Marketplace (1 day)

1. Enable Marketplace SDK
2. Create store listing
3. Upload all materials
4. Submit for review
5. Wait for approval (2-6 weeks)

### Step 4: Post-Submission (Ongoing)

1. Monitor for approval/rejection
2. Address any feedback quickly
3. Respond to user reviews
4. Update based on feedback
